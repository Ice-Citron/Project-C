# C Labs

Small buildable exercises for rebuilding C fluency.

Priorities:

- Write clear `.h` / `.c` boundaries.
- Make ownership rules explicit.
- Compile with warnings enabled.
- Run tests and sanitizers.
- Prefer small, boring APIs over clever implementations.

Initial labs:

- `dynamic-array`
- `ring-buffer`
- `sensor-log-parser`
- `hashmap`
- `finite-state-machine`
