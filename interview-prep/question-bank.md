# Question Bank

Use these as 40-minute drills. The goal is reasoning from first principles, not memorising standard architectures.

## General Systems

- Design a rate limiter for an API.
- Design a job queue for background tasks.
- Design a file upload service.
- Design a logging and alerting pipeline.
- Design a cache for expensive computations.

## Robotics-Adjacent

- Design a sensor data ingestion pipeline for a robot.
- Design a command queue for a robot controller.
- Design a safety monitor that stops a robot on invalid sensor data.
- Design a module that synchronises camera frames with IMU readings.
- Design a telemetry recorder that can survive process crashes.

## Pseudocode-Focused

- Given timestamped sensor readings, return the latest valid reading before a deadline.
- Given a stream of detections, suppress duplicates within a time window.
- Given producer/consumer rates, design a bounded buffer policy.
- Given robot states and events, implement a finite-state transition function.
- Given noisy measurements, maintain a rolling average with fixed memory.
